Lesson 02 · 读 ERC-20 合约

ABI · useReadContracts · decimals · multicall · 目标 30 分钟内跑通

为什么先学\"读合约\"？ Lesson 01 用 useBalance 读的是 原生代币（ETH/SepoliaETH），走 RPC 的 eth_getBalance。真实 DApp 90% 的读数据是 调合约：USDC 余额、NFT 持仓、DeFi 池子价格全是合约里的 view 函数。这一节把这一层打通，后面所有链上读操作都是同一个模式。

1. 前置准备（承接 Lesson 01）

  • Lesson 01 那一套还在：Node 20+、MetaMask、Sepolia 网络、有测试 ETH。
  • 推荐再领点 测试 LINK：faucets.chain.link/sepolia（Chainlink 官方水龙头，可以同时领 ETH + LINK，25 个 LINK 一次）。领完这一节演示看到的 balanceOf 就不是 0，效果直接。
  • 不领也能跑，只是余额显示 0，其他数据（name/symbol/decimals/totalSupply）照样能读——因为它们不看你的地址。

2. 三个必须先立住的概念

2.1 ABI 是什么

ABI = Application Binary Interface。人话：合约暴露给外部的接口说明书。前端拿着 ABI 才知道：

  • 合约有哪些方法、方法叫什么、要传什么参数、返回什么类型；
  • 方法是只读（view/pure）还是要发交易（nonpayable/payable）；
  • 合约会发哪些事件（下一节讲）。

ABI 通常是一坨 JSON，从 Etherscan 的合约页复制、或 Hardhat/Foundry 编译产物里拿。前端不需要塞整个 ABI——你要调哪些方法就只留哪些。ERC-20 我们这节只需要 5 个：name / symbol / decimals / totalSupply / balanceOf。

2.2 view 和 eth_call：读为什么不花 gas

标了 view（或 pure）的函数不上链、不广播、不花 gas。viem 把这类调用发成 eth_call：节点在本地跑一遍合约字节码、返回结果，没写状态所以不进区块。你把它想成"链上数据库的 SELECT"。

反过来，写操作（nonpayable/payable）必须签名 + 上链 + 花 gas，这是 Lesson 03 的事。

2.3 decimals：ERC-20 最容易踩的坑

链上所有代币金额都是整数（uint256）。真实"1 USDC" 存的是 1_000_000（6 位小数），真实"1 LINK" 存的是 1_000_000_000_000_000_000（18 位）。decimals() 告诉你要挪几位小数。

面试高频坑：看到 balanceOf 返回 5000000 就以为是 500 万 USDC——错，是 5 USDC。永远用 formatUnits(value, decimals)，不要用 formatEther（那是硬编码 18 位，USDC 会算错 12 个 0）。

4. 代码：三步定型

4.1 src/erc20.ts — 最小 ABI + 常量

// 只留我们要用的 5 个方法
export const erc20Abi = [
  { type: 'function', name: 'name',        stateMutability: 'view', inputs: [], outputs: [{ type: 'string' }] },
  { type: 'function', name: 'symbol',      stateMutability: 'view', inputs: [], outputs: [{ type: 'string' }] },
  { type: 'function', name: 'decimals',    stateMutability: 'view', inputs: [], outputs: [{ type: 'uint8'  }] },
  { type: 'function', name: 'totalSupply', stateMutability: 'view', inputs: [], outputs: [{ type: 'uint256' }] },
  { type: 'function', name: 'balanceOf',   stateMutability: 'view',
    inputs: [{ name: 'account', type: 'address' }],
    outputs: [{ type: 'uint256' }] },
] as const

export const SEPOLIA_TOKENS = {
  LINK: { address: '0x779877A7B0D9E8603169DdbD7836e478b4624789', label: 'LINK (Chainlink)' },
  WETH: { address: '0xfFf9976782d46CC05630D1f6eBAb18b2324d6B14', label: 'WETH' },
  USDC: { address: '0x1c7D4B196Cb0C7B01d743Fbc6116a902379C7238', label: 'USDC (Circle)' },
} as const
as const 是关键。加了它 TypeScript 才能从 ABI 推断出每个方法的入参和返回类型——不加你调 balanceOf 时是 any，加了就是 bigint。面试题：viem 为什么能给你零配置的强类型？靠 as const + TS 4.9+ 的字面量类型推断。

4.2 src/App.tsx — 一次 multicall 读全套

import { useState } from 'react'
import { useAccount, useReadContracts } from 'wagmi'
import { formatUnits } from 'viem'
import { erc20Abi, SEPOLIA_TOKENS, type TokenKey } from './erc20'

const [tokenKey, setTokenKey] = useState<TokenKey>('LINK')
const token = SEPOLIA_TOKENS[tokenKey]
const { address, isConnected } = useAccount()

const { data, isPending, isError, error, refetch } = useReadContracts({
  allowFailure: false,
  contracts: [
    { address: token.address, abi: erc20Abi, functionName: 'name' },
    { address: token.address, abi: erc20Abi, functionName: 'symbol' },
    { address: token.address, abi: erc20Abi, functionName: 'decimals' },
    { address: token.address, abi: erc20Abi, functionName: 'totalSupply' },
    { address: token.address, abi: erc20Abi, functionName: 'balanceOf',
      args: address ? [address] : undefined },
  ],
  query: { enabled: isConnected && !!address },
})

const [name, symbol, decimals, totalSupply, balance] = data ?? []

// 展示：formatUnits(balance, decimals) 才是人类可读金额

5. Multicall：一个必知的性能点

Naive 写法：调 5 个方法 = 5 次 eth_call = 5 次 RPC 往返 = 5 次 MetaMask/Infura 转发。慢，还容易触发免费 RPC 的限速。

useReadContracts 会把它们打包成一次到 Multicall3 合约（0xcA11bde05977b3631167028862bE2a173976CA11，几乎所有 EVM 链同地址部署）的 eth_call——合约在链上一次性把 5 个结果全返回给你。这是所有稍微像样的 DApp 前端都在用的技术，面试提到"你怎么优化 RPC 调用次数"这道题，就答这个。

6. 验证你的 win（3 分钟）

  • pnpm dev 起来后连上 MetaMask（Sepolia 网络）。
  • 下拉框选 LINK：应看到 name = ChainLink Token、symbol = LINK、decimals = 18、totalSupply 非零。
  • 如果领过 LINK，balanceOf 那一行显示的余额和 MetaMask 里"导入代币 LINK"后看到的应该完全一致——不一致就是 formatUnits 位数用错了。
  • 切到 USDC：注意 decimals = 6。你可以试着把 App.tsx 里的 formatUnits(balance, decimals) 强行改成 formatEther(balance)，看看 USDC 金额会离谱到什么程度——这就是那 12 个 0 的教训。
  • 断网重连，点「重新读一次」按钮验证 refetch()。

7. 20% 的原理，够撑面试第二层

  • ABI 编码：调 balanceOf(address) 时，viem 会把它编成 0x70a08231（函数选择器，是 keccak256("balanceOf(address)") 的前 4 字节）+ 32 字节地址。节点按 EVM 规则解码执行，把 uint256 编码回来。这一整套就叫 ABI 编码。
  • 为什么 balanceOf 返回 bigint 不是 number？ uint256 最大值是 2^256-1，JS 的 Number 最大安全整数只有 2^53-1。用 number 会精度爆炸（比如 USDT 大户余额直接被四舍五入丢钱）。
  • allowFailure 该怎么选？ 教学阶段用 false——任何一个失败就整体报错，好定位问题。生产里读一批 100 个 NFT 元数据时用 true，允许部分失败，避免一个坏合约拖垮整页。
  • 为什么 data 一开始是 undefined？ react-query 的取数模式。isPending 时还没数据，isSuccess 后 data 才有值。写 UI 时要么 data ?? [] 兜底，要么用 isPending 早返回。
