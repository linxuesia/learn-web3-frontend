Lesson 01 · 第一支能跑的 DApp

连接钱包 + 读地址 + 读原生代币余额 · 目标 25 分钟内跑通

为什么先学这个？ 一个月要出能演示的 DApp，最贵的第一步是 你手上真的有一个跑得起来的项目骨架。所有后续（发交易、调合约、读事件）都长在这个骨架上。别在这节纠结原理，先让它跑。

1. 前置准备（一次性，5 分钟）

  • 已装 Node 20+（node -v 看一下）。
  • Chrome/Brave/Arc 已装 MetaMask。
  • MetaMask 里切到 Sepolia 测试网（Settings → Advanced → 打开 Show test networks）。
  • 拿点 Sepolia 测试币：Google Cloud Faucet 或 PoW Faucet。0.05 ETH 够用一个月。

1.5 如何确认当前连的是哪个网络？（必备小技能）

MetaMask 有时候点了切换网络 UI 没刷新，容易让人误判。这一段以后会反复用——永远不要靠 UI 感觉判断当前在哪个网上，用下面这几种方式一秒验证。

方法 A：看 MetaMask 里的三处标识（最快）

  • 顶部网络下拉按钮：会写着 Sepolia / Ethereum Mainnet 等。
  • 余额那行的代币符号：主网显示 ETH，Sepolia 显示 SepoliaETH，Base Sepolia 显示 ETH（但网络名不同）。这行比顶部按钮更不容易骗人。
  • 底部胶囊按钮：同样写着当前网络名。

方法 B：Console 一行代码验（最硬核，面试也可能问）

打开任意正常网页（比如 sepolia.etherscan.io 或 ethereum.org——不能是 chrome:// 开头的内部页，MetaMask 不往那些页面注入），F12 → Console → 粘：

await window.ethereum.request({ method: 'eth_chainId' })

拿到十六进制的 chainId 后，对照下面这张表。或者一次查全套：

const chainId = await window.ethereum.request({ method: 'eth_chainId' })
const accounts = await window.ethereum.request({ method: 'eth_accounts' })
console.log({ chainId, chainIdDec: parseInt(chainId, 16), accounts })
坑点：如果报 Cannot read properties of undefined (reading 'request')，说明这个页面没被注入 window.ethereum——不是钱包坏了，是你在错的页面按 F12 了（新标签页、chrome://extensions、about:blank 都不会被注入）。换个普通网站再试。

方法 C：chainlist.org 直连测试

打开 chainlist.org（勾上 Testnets）→ 点 "Connect Wallet"，它直接把当前网络亮出来，还能一键给 MetaMask 加新链。

2. 起一个空的 Vite + React + TS 项目（3 分钟）

cd ~/sia
npm create vite@latest web3-hello -- --template react-ts
cd web3-hello
npm i wagmi viem @tanstack/react-query
npm run dev

浏览器打开 http://localhost:5173，看到 Vite 默认页 = OK。

3. 配置 wagmi（5 分钟）

建 src/wagmi.ts：

import { http, createConfig } from 'wagmi'
import { sepolia } from 'wagmi/chains'
import { injected } from 'wagmi/connectors'

export const config = createConfig({
  chains: [sepolia],
  connectors: [injected()],
  transports: {
    [sepolia.id]: http(),
  },
})

改 src/main.tsx，把 App 包在 wagmi + react-query 的 Provider 里：

import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import { WagmiProvider } from 'wagmi'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import { config } from './wagmi'
import App from './App'
import './index.css'

const queryClient = new QueryClient()

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <WagmiProvider config={config}>
      <QueryClientProvider client={queryClient}>
        <App />
      </QueryClientProvider>
    </WagmiProvider>
  </StrictMode>,
)

4. 写「连钱包 + 读余额」的 UI（8 分钟）

把 src/App.tsx 换成：

import {
  useAccount, useConnect, useDisconnect, useBalance, useChainId,
} from 'wagmi'
import { formatEther } from 'viem'

export default function App() {
  const { address, isConnected } = useAccount()
  const chainId = useChainId()
  const { connectors, connect, status, error } = useConnect()
  const { disconnect } = useDisconnect()
  const { data: balance } = useBalance({ address })

  return (
    <main style={{ maxWidth: 640, margin: '3rem auto', fontFamily: 'system-ui' }}>
      <h1>Hello Web3</h1>

      {!isConnected ? (
        <div>
          {connectors.map((c) => (
            <button key={c.uid} onClick={() => connect({ connector: c })}>
              连接 {c.name}
            </button>
          ))}
          {status === 'pending' && <p>连接中…</p>}
          {error && <p style={{ color: 'crimson' }}>{error.message}</p>}
        </div>
      ) : (
        <div>
          <p>地址：<code>{address}</code></p>
          <p>当前链 ID：{chainId}</p>
          <p>
            余额：
            {balance
              ? \`\${formatEther(balance.value)} \${balance.symbol}\`
              : '读取中…'}
          </p>
          <button onClick={() => disconnect()}>断开</button>
        </div>
      )}
    </main>
  )
}

5. 验证你的 win（2 分钟）

  • 点「连接 Injected / MetaMask」→ MetaMask 弹窗 → 授权。
  • 页面上应出现你的地址、Chain ID = 11155111（Sepolia）、Sepolia ETH 余额。
  • 断开后按钮变回「连接」。

如果这几步都成立，恭喜——你已经跑通了 web3 前端的最短闭环：钱包 → 前端 → 链 → 数据 → UI。所有后续内容都是在这条链路上加东西。

6. 20% 的原理，够撑面试第一层

  • 钱包为什么能被网页读到？ 因为浏览器钱包会往页面注入一个符合 EIP-1193 的 provider（早期是 window.ethereum，现代是 EIP-6963 多钱包发现）。wagmi 的 injected() 就是在这层之上做的封装。
  • viem vs ethers vs wagmi？ viem 是底层 EVM 客户端；wagmi 是它的 React hooks 封装；ethers 是老前辈，很多老项目还在用，但新项目 2024 起主流是 wagmi + viem。
  • 为什么要 react-query？ wagmi 的 hooks 底层用 react-query 做缓存和 refetch，链上数据本质是"网络请求"，缓存/失效逻辑复用 react-query 是最省心的方案。
